## Performance Insights

1. **INSERT Performance Excellence**
   - Prepared statements are 42% faster than regular INSERTs
   - Small batches (10 rows) maintain high throughput
   - Performance degrades significantly with large batches

2. **SELECT Performance Adequate**
   - ~1,500-1,700 queries/second for basic operations
   - Range queries slightly faster than simple lookups
   - High memory allocation count suggests optimization potential

3. **Batch Scaling Issues**
   - 10‚Üí100 rows: 6.3x slower (expected: ~10x)
   - 100‚Üí1000 rows: 21.6x slower (expected: ~10x)
   - Non-linear scaling indicates algorithmic bottleneck

## Optimization Opportunities

Priority 1: Reduce allocations in SELECT queries (target: <1000 allocs/op)
Priority 2: Improve batch scaling for 1000+ row operations
Priority 3: Implement memory pooling for frequently allocated objects
