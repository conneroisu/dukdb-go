## Overview

**dukdb-go** is a pure-Go implementation of the DuckDB database driver that eliminates CGO dependencies while maintaining full compatibility with Go's `database/sql` interface. Built using the `purego` library for Foreign Function Interface (FFI), it enables cross-platform deployment with static binaries.

## Quick Start

```go
package main

import (
    "database/sql"
    "log"
    
    _ "github.com/connerohnesorge/dukdb-go/driver"
)

func main() {
    db, err := sql.Open("duckdb", ":memory:")
    if err != nil {
        log.Fatal(err)
    }
    defer db.Close()
    
    rows, err := db.Query("SELECT 42 as answer, 'Hello DuckDB' as greeting")
    if err != nil {
        log.Fatal(err)
    }
    defer rows.Close()
    
    for rows.Next() {
        var answer int
        var greeting string
        err := rows.Scan(&answer, &greeting)
        if err != nil {
            log.Fatal(err)
        }
        log.Printf("Answer: %d, Greeting: %s", answer, greeting)
    }
}
```

## Architecture

This driver uses a progressive enhancement approach:

1. **FFI Layer**: `purego` provides CGO-free C function calls
2. **Driver Interface**: Standard `database/sql/driver` implementation
3. **Type System**: Comprehensive support for all DuckDB data types
4. **Optimization**: Connection pooling, statement caching, and context support
